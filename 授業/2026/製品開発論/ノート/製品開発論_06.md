# 第6回：コンセプト開発

## 目標

製品開発の骨組みとなるコンセプトを生み出すための考え方を考える。
そのために、改めてコンセプトとはどういう概念なのかを理解する。

## コンセプトとは

その製品が、お客様に対して提供する特別の価値や体験を端的に表現したもの。
（コンセプトは端的であればよく、詩的であったりカッコよかったりする必要はない）

（余談）スタバのコンセプトは、なぜ知られている？
→ ビジネス雑誌の取材により広まったから。

## アイデアがありきたりな商品に終わる理由

課題とアイデアが1対1になってしまっているから。

### 悪い例

サッカーチームの活性化

- 地元が応援に来ない → 地元の人に優待券を配る
- スポンサー不足 → スポンサー量の減額
- 応援グッズがない → 応援グッズを開発する


筋が通っているわけではなく（コンセプトがなく）、問題に対してただ1つずつ解決していってるだけ。
「課題 → アイデア」ではなく、==「課題 → コンセプト → アイデア」==とすると、他にはないアイデアで差別化できるようになったり、アイデアを発掘できるようになったりする。

（日本の自治体は「課題 → アイデア」になっている）

### 良い例

シント＝トロイデン
ベルギーのプロサッカーチーム。オーナーはDMM.com。

コンセプトとアイデア

- 日本サッカーの欧州拠点化
	- 若き才能を応援する方法
		- 露出よりも影から応援する
			- 名刺にスポンサー名を入れるだけ
			- 本当に若者を応援していることが伝わり、会社側にもメリットがある
		- 結果よりもプロセス
		- ビッグクラブへの積極的な選手譲渡
			- 立派に育ってもらったら、残ってもらう必要はない
			- 移籍金をもらって、チームを潤わせる
- グローバルな健全経営

### 考える手順

1. インプット（素材）
2. シンセンス（統合）
3. コンセプト（骨格となる発想）

## コンセプトを開発するステップ

### 1. 方向性の決定

#### コンセプトの種となる視点を決める4つのアプローチ

1. ポジティブアプローチ
	- 長所を伸ばす。
2. ネガティブアプローチ
	- 短所を補う。
3. ギャップアプローチ
	- 他との差を埋める。
4. ビジョンアプローチ
	- 理想に近づける。

#### 例）カレーの会社

1. ポジティブアプローチ
	- 着眼点
		- スパイスが得意である
	- 結論
		- 他にはないスパイス、独特の調合を研究
	- ==思考法==
		- 自社が持っている長所を最大化し、圧倒的な武器へと昇華させる
2. ネガティブアプローチ
	- 着眼点
		- 同社の弱点は大量生産である。
	- 結論
		- 大量生産技術を獲得、強化する。
	- ==思考法==
		- マイナスをゼロに近づけ、全体のボトルネック（弱点）をなくす。
3. ギャップアプローチ
	- 着眼点
		- ライバルは量販店で非常に売れている。
		- 高級と低価格の二極化で、中間ゾーンが不在。
	- 結論
		- 量販店が求めているカレーを開発する。
		- 注価格帯のカレーを開発する。
	- ==思考法==
		- 明確な比較対象（競合）を決定し、その差分を埋めることで競争力を保つ。
		- 社会（顧客）ニーズと現状のギャップ（空白領域）を狙い撃ちする。
4. ビジョンアプローチ
	- 着眼点
		- 同社は家族の幸せの瞬間に、自社のカレーを食べてほしいという願いがある。
	- 結論
		- 家族が一緒に楽しめるカレーを開発する。
	- ==思考法==
		- 得意不得意や他者との比較は無視し、純粋な内発的動機から逆算する。

#### ライバルとの差別化

- 「ネガティブアプローチ」と「ギャップアプローチ」
	- 取り組みやすいが、ライバルも同じ弱点や市場の隙間を見ているため、結果的に横並びになる。（同質化を招きやすい。）
	- 日本企業が特に選択肢、利益率が低くなっている。
- 「ポジティブアプローチ」と「ビジョンアプローチ」
	- 自社固有の強みや理想が起点のため、ライバルに模倣できない可能性。
	- 他社にはない強み、目指す方向性が明確な企業や企画者が採用。

### 2. 情報の統合・探索

集めた情報を調理し、新たな視座を見つける、4つの思考モード。

#### 4つの思考モード

1. 俯瞰
	- 視野を広げ、隠れた競合を発見。
	- 視座を高くし、可能性を横に広げる。
	- 例）ノートの会社
		- ルーズリーフも、カメラでの撮影も、「ノート」であると言える。
	- 例）女子高生がCDを買わなくなった理由
		- 携帯電話の普及
2. 深掘り
	- なぜを繰り返し、根源的な欲望を探る。
	- 顧客ニーズの本質を、縦に広げる。
	- 例）25歳、TOEICを受け続ける女性
		- 属性・機能的価値
			- 自分の英語能力を客観化したい。
		- 情緒的価値
			- 英語を使いこなす、かっこいい自分。
		- 社会・生活価値（本音）
			- 派遣社員として働いている、就活に失敗した過去の自分をリセットしたい。
3. 分類
	- 複雑なものを整理し、市場に分割する
	- 多様な軸の組み合わせで、新しい領域を探す。
	- 例）パソコン
		- 軸の組み合わせで整理する
			- 本格的・がっつりな仕事 ↔️ 簡単・手軽な仕事
			- 据え置き ↔️ 持ち運び
			- お客さんがどの軸で（価値などを）分類しているかで考える
		- 
4. 混合
	- 無関係なものを掛け合わせ、新結合をうむ。
