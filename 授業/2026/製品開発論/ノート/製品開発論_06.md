# 第6回：コンセプト開発

## 目標

製品開発の骨組みとなるコンセプトを生み出すための考え方を考える。
そのために、改めてコンセプトとはどういう概念なのかを理解する。

## コンセプトとは

その製品が、お客様に対して提供する特別の価値や体験を端的に表現したもの。
（コンセプトは端的であればよく、詩的であったりカッコよかったりする必要はない）

（余談）スタバのコンセプトは、なぜ知られている？
→ ビジネス雑誌の取材により広まったから。

## アイデアがありきたりな商品に終わる理由

課題とアイデアが1対1になってしまっているから。

### 悪い例

サッカーチームの活性化

- 地元が応援に来ない → 地元の人に優待券を配る
- スポンサー不足 → スポンサー量の減額
- 応援グッズがない → 応援グッズを開発する


筋が通っているわけではなく（コンセプトがなく）、問題に対してただ1つずつ解決していってるだけ。
「課題 → アイデア」ではなく、==「課題 → コンセプト → アイデア」==とすると、他にはないアイデアで差別化できるようになったり、アイデアを発掘できるようになったりする。

（日本の自治体は「課題 → アイデア」になっている）


### 良い例

ベルギーのプロサッカーチーム。オーナーはDMM.com。

 コンセプトとアイデア

- 日本サッカーの欧州拠点化
	- 若き才能を応援する方法
	- 
- グローバルな健全経営
